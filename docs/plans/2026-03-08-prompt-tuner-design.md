# 提示词自动调优工具设计文档

## 1. 项目概述

### 1.1 背景
车载停车场景的手机通知汇总智能助手，需要优化提示词以使模型输出符合TTS语音播报和车机屏幕展示的要求。

### 1.2 目标
创建一个Python CLI工具，能够：
- 调用智谱API测试不同模型的提示词效果
- 自动评估输出质量并优化提示词
- 支持5-8小时无人值守运行
- 生成详细的优化报告

### 1.3 核心指标
- 质量收敛阈值: 总分 >= 85, 各维度 >= 15
- 稳定性验证: 连续3次总分 >= 80
- 最大迭代次数: 每个模型50次
- 最大运行时间: 4小时（可配置）

### 2.1 执行流程
1. 加载初始提示词和测试数据
2. 对每个配置的模型执行测试
3. 解析输出并评分（0-100分）
4. 如果分数 < 阈值， 分析问题并优化提示词
5. 重复直到达到目标或最大迭代次数
6. 生成最终报告

---

## 3. 评分系统 (总分100分)

| 维度 | 分值 | 检查项 |
|------|------|--------|
| 格式正确性 | 20 | 必须包含【思考】...【思考】...<show>...</show>；标签完整；无额外内容 |
| 内容合规性 | 20 | 无markdown；无特殊符号；无编造信息 |
| 分级准确性 | 20 | 一级/二级/三级分类正确；去重处理正确 |
| TTS约束 | 20 | 不超过15句；单句不超过30字；口语化；三级不播报 |
| SHOW约束 | 20 | 按优先级排序；单条不超过20字；无重复 |

**收敛条件**:
- 总分 >= 85 且 每个维度 >= 15
- 或连续3次总分 >= 80（稳定性验证）

## 5. 提示词优化策略

### 5.1 问题诊断规则
| 问题类型 | 优化策略 |
|----------|----------|
| 格式错误 | 强化格式约束，增加正确示例 |
| 输出英文 | 添加"禁止英文输出"规则 |
| 分级错误 | 补充分级示例，明确边界 |
| TTS过长 | 强化句数限制，强调简洁 |
| SHOW混乱 | 强调结构化格式，添加示例 |
| 娡型不稳定 | 添加"必须严格遵循"强调 |

### 5.2 优化优先级
1. 先修复格式问题（最高优先级）
2. 再修复内容合规性
3. 最后优化分级准确性
